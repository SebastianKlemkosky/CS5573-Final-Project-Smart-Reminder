How to update PythonAnywhere after GitHub changes
-------------------------------------------------
In a PythonAnywhere Bash console, run:

cd ~/CS5573-Final-Project-Smart-Reminder
git reset --hard HEAD
git pull

Then:
- go to the Web tab
- click Reload

Why use git reset --hard HEAD first?
------------------------------------
Sometimes PythonAnywhere has local changes to app.py or other files.
That can block git pull.
Running git reset --hard HEAD clears those tracked local changes so the server matches GitHub.

Warning:
This deletes tracked local changes on PythonAnywhere.
Do not run it if you made important edits directly on PythonAnywhere that are not in GitHub.

If git pull fails because of local changes
------------------------------------------
Option 1: safest, inspect first
git status
git diff app.py

Option 2: overwrite server copy with GitHub version
git reset --hard HEAD
git pull
